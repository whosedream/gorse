# 电商 AI 智能导购重排系统：技术规格说明书与软件设计文档

## 一、 系统拓扑与双轨流量边界

系统整体采用非对称快慢轨流水线架构，通过解耦算力与存储，实现高并发极速响应与深度智能推理的有机统筹。

```
                       [ 客户端导购请求 ]
                                │
                                ▼
  ┌───────────────────────【 同步快轨 】───────────────────────┐
  │                                                            │
  │  1. 接收网关请求                                           │
  │  2. FSM 零反射流式解析内存数据                             │
  │  3. 调取高性能缓存（本地预热特征）                         │
  │  4. ants 协程池多路并行提取与打分                           │
  │  5. 25毫秒内输出商品重排列表 ───► [ 吐出结果给用户 ]         │
  │                                                            │
  └────────────────────────────┬───────────────────────────────┘
                               │
                               │ (全域会话、点击、加购行为)
                               ▼
  ┌───────────────────────【 异步慢轨 】───────────────────────┐
  │                                                            │
  │  1. 高性能消息队列（Kafka）流式泵送                         │
  │  2. 慢轨 Agent 框架（对话状态机工作流）                     │
  │  3. 调度大模型（输入缓存复用）进行思维链推理                 │
  │  4. 产出高维意图表征与特征权重                              │
  │  5. 核查-反馈-修正-复核（防感知漂移控制）                  │
  │  6. 异步更新至在线特征库（为快轨热预热）                     │
  │                                                            │
  └────────────────────────────────────────────────────────────┘

```

## 二、 同步快轨精密设计（Fast-Track Detailed Design）

同步快轨的目标是确保在线精排与重排的单机吞吐量 $\ge$ 20,000 QPS，且 P99 延迟 $\le$ 25毫秒。核心组件采用防御式内存复用设计。

### 1. 手写有限状态机流式解析器（/pkg/fsm）

* **设计原理**：彻底抛弃标准库的运行时反射机制，通过预加载的运行时类型描述构建轻量级状态转移图。逐字节扫描输入流，直接就地将解析出的关键字段（如会话标识、槽位、特征版本戳）写入目标重排结构体的预留缓冲区中。
* **技术规格**：
* 单次解析堆分配次数：0 次。
* 异常处理：包含显式的非阻塞错误退避分支，遇到格式破损的输入数据时立即报错跳过，并激活本地默认推荐特征，严禁阻塞核心调度。



### 2. 无锁本地内存池（/pkg/pool/memory）

* **设计原理**：利用 `sync.Pool` 为高频创建的解析上下文、重排打分切片提供对象复用。深度融合 Go 语言底层的调度模型，每个逻辑处理器（P）维护独立的本地池与共享双端队列，最大限度消除多协程下的锁竞争。
* **技术规格**：
* **生命周期约束**：对象捞出（Get）后立即执行字段置空；归还（Put）前必须在延迟函数中执行彻底的重置，清空缓冲区长度并释放底层引用，严禁产生跨请求的数据交叉污染。
* **逃逸防范**：核心计算逻辑禁止将池内局部对象以指针形式作为返回值，所有打分结果必须就地在传入的引用中填充，通过静态编译标记强制约束编译器将生命周期锁定在栈区。



### 3. 自适应有界并发协程池（/pkg/pool/goroutine）

* **设计原理**：限制工作协程数上限，通过带有固定物理容量的阻塞通道管理任务积压量。
* **技术规格**：
* **动态伸缩**：当队列任务水位达到 60% 阈值时自动扩容协程；系统空闲后超时释放。
* **显式背压**：当突发秒杀洪峰导致有界队列完全饱和时，提交任务的分支采用非阻塞结构，拒绝无限期挂起，立即向上游网关返回过载降级错误。
* **并行同步控制**：多路特征拉取采用带超时控制的上下文绑定。任何一路子任务发生严重超时或报错，立即触发快速失败机制，向其余并行分支发送取消信号，终止无谓的网络输入输出与算力损耗。

## 三、 异步慢轨意图决策与一致性控制（Slow-Track Detailed Design）

异步慢轨通过大模型思维链推理捕捉用户长周期跨域动态兴趣。其平均耗时在 150毫秒至800毫秒 之间。

### 1. 慢轨智能体工作流（/pkg/agent）

* **设计原理**：基于 Go 原生的对话状态机与有向无环图工作流编排大模型的工具调用。
* **技术规格**：
* **算力优化**：深度集成大模型的**输入缓存复用**特性。针对长周期多轮会话和长特征向量，通过缓存公共上下文，将重复检索的账单开销和时延降到最低。



### 2. 防感知漂移一致性流水线（/internal/rk/anti_drift）

* **设计原理**：由于大模型推理存在非对称延迟，必须防止慢轨计算结果覆盖用户在快轨端产生的最新即时行为，建立“核查-反馈-修正-复核”闭环控制链。
* **技术规格**：
* **核查**：慢轨输出的特征必须强制携带计算启动时的基线时间戳。
* **反馈与修正**：写入特征库时，比对当前最新时间戳。若检测到感知漂移指标超出安全阈值，启动动态融合算法，采用衰减因子 $\alpha$ 对慢轨深层意图特征与快轨最新交互特征进行加权合并，严禁盲目覆盖。
* **复核**：后台建立双轨台账复盘机制，定期对用户长周期静态偏好与即时槽位特征执行差异复盘，动态微调衰减系数，控制意图表征感知漂移偏差 $\le$ 5%。
* **优雅降级**：慢轨服务超时或算力耗尽时执行算力熔断，完整保留用户历史记忆空间，平滑退避至预先蒸馏的高阶精排小模型进行打分，确保“算力可断，记忆不断”。

## 四、 核心数据契约与接口定义

快慢轨之间、以及与在线特征库之间的通信采用精简、明确的数据契约，杜绝冗余字段。

### 1. 同步快轨请求契约（JSON 字节流拓扑）

```json
{
  "session_id": "string (固定长度 UUID)",
  "version_stamp": "int64 (毫秒级时间戳)",
  "slots": {
    "category": "string",
    "brand": "string"
  },
  "candidate_ids": ["int64 (商品ID数组)"]
}

```

### 2. 慢轨意图特征更新契约（内部台账结构）

```go
type IntentFeatureUpdate struct {
	SessionID        string    // 会话唯一标识
	BaselineVersion  int64     // 慢轨计算启动时的快轨行为时间戳
	IntentVector     []float32 // 高维语义意图特征向量
	CategoryWeights  map[string]float32 // 类目动态权重偏好
	DriftThreshold   float32   // 感知漂移安全阈值
}

```

## 技术栈准入白名单（追加至设计文档）

所有智能体（包括驾驶员与打字员）在编写、重构或引入外部依赖时，必须严格限制在以下白名单范围内，严禁擅自引入未经声明的第三方依赖库：

| 架构分层 | 组件分类 | 官方核定选型（白名单） | 工业级硬性约束边界 |
| --- | --- | --- | --- |
| **底座核心** | 开发语言 | **Go** | 强制采用原生底层特性开发，严禁在快轨主路径引入运行时反射。 |
| **快轨组件** | 协程并发 | **ants / go-adaptive-pool** | 必须具备有界队列拓扑、动态伸缩能力以及非阻塞背压限流机制。 |
| **快轨组件** | 协议解析 | **Gojay / jsoniter** | 必须采用无反射的有限状态机或边读边转的迭代器模式，主路径内存分配必须为零。 |
| **快轨组件** | 任务同步 | **sync/errgroup** | 必须绑定带有超时控制的上下文，一旦子任务异常立即触发快速失败。 |
| **快轨组件** | 内存复用 | **sync.Pool** | 必须结合底层调度模型实现本地无锁复用，放回对象池前必须执行彻底的清空重置。 |
| **系统保护** | 线上熔断 | **sony/gobreaker / sentinel-golang** | 轻量级重排路径优先选用前者；对接大规模分布式流控生态时选用后者。 |
| **中间件** | 消息队列 | **Kafka** | 用于异步慢轨行为数据的流式高吞吐泵送，必须配置合理的批量提交策略。 |
| **存储层** | 本地缓存 | **Redis / Aerospike** | 内存级超低延迟热特征存储，快轨读取耗时必须死守在微秒级别。 |
| **慢轨 AI** | 基座大模型 | **DeepSeek V4 Pro** | 1.6万亿参数混合专家模型，必须激活输入缓存复用特性以对冲上下文重读成本。 |
| **慢轨 AI** | 智能体框架 | **only-ai/chatcore / 自研工作流** | 拒绝 Python 重型框架，必须使用 Go 原生状态机或有向无环图执行引擎开展轻量化编排。 |
